# AGENTS.md

이 파일은 Codex가 이 프로젝트를 순차적으로 진행할 때 따를 실행 계획입니다.

## 원칙

- 공식 Meta Instagram API로 가능한 자동화만 구현한다.
- 브라우저 조작, 비공식 API, 세션 탈취 방식은 사용하지 않는다.
- 사용자의 상호작용 없이 선제 대량 DM을 보내는 기능은 만들지 않는다.
- 관리자 화면은 실제 운영 콘솔을 우선하고, 마케팅 랜딩은 후순위로 둔다.
- 각 단계는 실행 가능한 결과물을 남기고 검증한다.
- 기능 개발 또는 고도화 작업 완료 시 지정된 원격 저장소(https://github.com/angibeom0985-arch/auto_dm)에 커밋 및 푸시(Push)를 수행한다.
- 배포 호스팅은 Vercel 환경을 고려하여 구성하고 프론트엔드/백엔드 주소를 환경 변수로 관리한다.

## 1단계: 웹 콘솔 MVP

- Vite + React + TypeScript 프로젝트 구성
- 자동화 대시보드 UI 구현
- 댓글 키워드 DM 플로우 빌더 UI 구현
- 이벤트 로그와 런칭 준비 체크 UI 구현
- 반응형 레이아웃 검증

완료 기준:

- `npm run dev`로 로컬 실행 가능
- `npm run build` 통과
- 첫 화면에서 서비스의 핵심 업무를 바로 조작할 수 있음

## 2단계: 백엔드 설계

- API 서버 구조 선택
- 데이터베이스 스키마 작성
- 계정, 자동화, 메시지, 이벤트, 발송 로그 모델 정의
- 환경변수 목록 작성

완료 기준:

- DB 마이그레이션 초안 생성
- 로컬에서 API 서버 실행 가능
- 프론트엔드가 목업 대신 API 데이터를 읽을 준비 완료

## 3단계: Meta OAuth 연결

- Meta Login 또는 Instagram Login 방식 결정
- OAuth 시작/콜백 API 구현
- Page 및 Instagram 계정 목록 조회
- access token 암호화 저장
- 토큰 만료일과 재연결 상태 표시

완료 기준:

- 테스트 계정으로 Instagram professional account 연결 가능
- 연결된 계정이 대시보드에 표시됨

## 4단계: Webhook 수신

- Webhook verification GET 구현
- Webhook POST 수신 구현
- X-Hub-Signature 검증
- 댓글, DM, postback 이벤트 저장
- 중복 이벤트 방지

완료 기준:

- Meta App Dashboard의 테스트 Webhook 수신 성공
- 실제 테스트 댓글 또는 DM 이벤트 저장 성공

## 5단계: 자동화 엔진

- 트리거 매칭 엔진 구현
- 댓글 키워드 매칭
- DM 키워드 매칭
- 자동화 활성/비활성 상태 반영
- 메시지 템플릿 렌더링

완료 기준:

- 저장된 이벤트가 자동화 규칙과 매칭됨
- 발송 후보 작업이 큐에 등록됨

## 6단계: 발송 큐와 Meta API

- BullMQ 또는 동등한 큐 도입
- Private Reply 발송 구현
- Instagram Send API 발송 구현
- rate limit, retry, backoff 적용
- 실패 사유 저장

완료 기준:

- 댓글 키워드에 대한 Private Reply 발송 성공
- 사용자가 보낸 DM에 자동 응답 성공
- 실패 로그가 운영 화면에 표시됨

## 7단계: 운영 기능

- 자동화 생성/수정/삭제
- 메시지 템플릿 관리
- 리드 목록
- 대화 상태
- 발송량 통계
- 계정별 사용량 제한

완료 기준:

- 비개발자가 자동화 플로우를 만들고 끌 수 있음
- 발송 결과와 오류를 화면에서 확인 가능

## 8단계: 런칭 준비

- 인증과 권한 분리
- 결제 플랜 적용
- 개인정보 삭제 요청 처리
- 로그 보관 정책 적용
- 배포 환경 구성
- Meta App Review 자료 정리

완료 기준:

- 베타 고객이 실제 Instagram 계정을 연결해 사용할 수 있음
- 운영자가 정책 위반 가능성을 모니터링할 수 있음
