# 사용자가 직접 해야 하는 실무 & Meta API 설정 가이드 (USER_TODO)

이 문서는 개발자가 백엔드 및 자동화 엔진을 완성한 후, **실제 Meta(Facebook) 및 Instagram 공식 API를 연동하여 서비스를 작동시키기 위해 대표님(사용자)이 직접 수행해야 하는 업무와 설정 방법**을 단계별로 설명합니다.

---

## 1단계: Instagram 프로페셔널 계정 전환 및 Facebook 연동
Meta API는 개인 인스타그램 계정에서는 작동하지 않으며, 반드시 **비즈니스(또는 크리에이터) 계정**이어야 하고 **페이스북 페이지와 연결**되어야 합니다.

### 1-1. Instagram 계정 프로페셔널 전환 (스마트폰 앱)
1. 인스타그램 앱 실행 후 **프로필 -> 우측 상단 메뉴(줄 3개) -> 설정 및 개인정보**로 이동합니다.
2. 아래로 스크롤하여 **계정 유형 및 도구**를 선택합니다.
3. **프로페셔널 계정으로 전환**을 누르고, 카테고리(예: 크리에이터, 쇼핑몰 등)를 선택한 뒤 **비즈니스** 또는 **크리에이터**를 선택합니다.
4. **메시지 설정**: 설정 중 "메시지 관리 권한 허용" 팝업이 뜨면 반드시 **허용(ON)**해 줍니다. 
   - *놓쳤다면*: 프로필 -> 메시지 및 스토리 답장 -> 메시지 관리 권한 -> **메시지 액세스 허용을 활성화(ON)**합니다. (이 작업이 없으면 Webhook이 작동하지 않습니다.)

### 1-2. Facebook Page 생성 및 Instagram 연동
1. PC 브라우저로 Facebook에 로그인한 뒤, 좌측 메뉴에서 **페이지 -> 새 페이지 만들기**를 눌러 비즈니스용 Facebook Page를 생성합니다.
2. 생성된 페이지의 **설정 -> 연결된 계정 -> Instagram** 메뉴로 이동합니다.
3. **계정 연결** 버튼을 누르고, 앞서 프로페셔널로 전환한 인스타그램 계정으로 로그인하여 연결을 완료합니다.

## 2단계: Meta for Developers 앱 생성 및 기본 설정
실제 사용자가 로그인하여 액세스 토큰을 발급받을 수 있는 API 게이트웨이를 구축하는 단계입니다.

### 2-1. Meta 개발자 계정 등록 및 앱 생성
1. [Meta for Developers](https://developers.facebook.com/)에 접속하여 가입 및 비즈니스 인증을 진행합니다.
2. 우측 상단 **내 앱 -> 앱 만들기**를 누릅니다.
3. 앱 유형 선택 화면에서 **"기타"** 또는 **"비즈니스"**를 선택합니다. (인스타그램 메시징 권한을 쓰려면 반드시 비즈니스 또는 소비자 유형이어야 합니다. 일반적으로 **비즈니스** 추천)
4. 앱의 표시 이름을 입력하고 연락처 이메일을 기입한 뒤 앱을 생성합니다.

### 2-2. Instagram Graph API 제품 추가
1. 앱 대시보드로 이동한 뒤, 제품 추가 리스트에서 **Instagram Graph API**와 **Facebook 로그인(Facebook Login for Business)**의 **설정** 버튼을 눌러 앱에 추가합니다.

## 3단계: Webhook 및 리디렉션 URL 설정
웹 콘솔의 설정 탭에 입력했던 Webhook URL과 OAuth 설정을 Meta 앱 대시보드에 실제로 동기화하는 단계입니다.

### 3-1. Webhook 설정
1. Meta 앱 대시보드 좌측 메뉴에서 **Webhooks**를 선택합니다.
2. 상단 드롭다운에서 **Instagram**을 선택하고 **이 개체 구독하기**를 클릭합니다.
3. **Callback URL**에 백엔드 주소(`https://instagram.gowith153.com/api/webhook/instagram`)를 입력합니다.
4. **Verify Token**에 백엔드 `.env`에 정의한 `META_VERIFY_TOKEN` 문자열을 기입하고 **확인 및 저장**을 누릅니다. (백엔드가 정상 응답하면 연동이 완료됩니다.)
5. 구독 필드 목록에서 **comments**(댓글 반응용)와 **messages**(DM 반응용)의 **구독** 버튼을 클릭합니다.

### 3-2. OAuth 리디렉션 URL 설정
1. 좌측 메뉴 **Facebook 로그인 -> 설정**으로 이동합니다.
2. **유효한 OAuth 리디렉션 URI** 란에 백엔드의 콜백 주소(`https://instagram.gowith153.com/api/auth/facebook/callback`)를 추가하고 변경사항을 저장합니다.

## 4단계: Meta App Review (앱 검수) 통과를 위한 준비
개발 완료 후 앱을 라이브(Live) 모드로 전환하려면, Meta의 검수를 받아 공식적인 메시징 권한을 획득해야 합니다. 개발 단계에서는 등록된 테스트 계정에서만 자동화가 작동합니다.

### 4-1. 필수 제출 권한 목록
앱 검수 신청 시 다음 4가지 권한을 선택하여 검수를 신청해야 합니다:
- `instagram_business_basic`: 인스타그램 계정 프로필 정보 조회
- `instagram_business_manage_messages`: 사용자가 보낸 DM에 답장 발송
- `instagram_business_manage_comments`: 릴스/피드에 달린 댓글 감지 및 Private Reply 전송
- `pages_show_list` & `pages_read_engagement`: 연결된 페이스북 페이지 리스트 확인

### 4-2. 검수용 데모 영상 제작 방법
Meta 검수관은 시스템이 실제로 스팸 정책을 위반하지 않고 공식 정책 내에서 동작하는지 영상으로 확인합니다.
1. **시나리오**: 모의 유저가 릴스 댓글에 '자료'를 적었을 때, 시스템이 이를 감지하여 해당 유저의 DM으로 다운로드 링크 가이드를 보내는 화면을 화면 녹화합니다.
2. **필수 포함 요소**:
   - 페이스북 로그인(OAuth)을 클릭해 권한 동의 팝업창이 뜨는 과정
   - 릴스나 피드에 댓글을 다는 모바일 화면
   - 대시보드의 실시간 로그에 댓글이 정상 감지되어 찍히는 화면
   - 인스타 DM 창을 열어, 1회성 답장 메시지가 정상 도착한 결과 확인

## 5단계: 서비스 약관 및 개인정보 처리방침 배포
Meta 앱 검수 및 Facebook 로그인 연동을 활성화하기 위해서는 도메인과 정책 문서가 필수적으로 웹상에 존재해야 합니다.

1. **개인정보 처리방침(Privacy Policy URL)**: 사용자의 인스타그램 토큰, 유입 리드 정보 등을 안전하게 보관하며, 사용자가 원할 시 즉시 삭제할 수 있음을 명시한 정책 페이지를 웹사이트 상에 제작하여 업로드합니다.
2. **데이터 삭제 요청 콜백 URL**: 사용자가 페이스북 설정에서 앱 연동 해제 및 데이터 삭제를 누를 시, Meta가 호출할 API 엔드포인트(`https://instagram.gowith153.com/api/auth/deauthorize`)를 구현 및 배포하여 등록해 줍니다.
